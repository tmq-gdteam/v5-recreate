# customers.new App Brief

## App Goal

Create a modern, clean, premium customer intelligence and engagement web app for sales, support, and marketing teams.

The app should help teams:

* monitor customer activity
* manage conversations
* analyze user journeys
* enrich visitor/contact data
* deploy and manage AI chat widgets
* train AI assistants with business knowledge

All within one unified workspace.

---

## Audience

* SaaS companies
* Sales teams
* Customer success teams
* Support teams
* Growth and marketing teams
* Operators who need fast, organized dashboards for customer data and conversations

---

## Core App Style

* Left sidebar navigation
* Top utility bar (search, notifications, help, profile)
* Modular card-based layout
* Clean analytics presentation
* Compact but readable tables
* Structured side panels and drawers
* Status badges, chips, and filters
* Fully responsive across desktop, tablet, and mobile

---

## Pages to Build

### 1. Dashboard

**Sections:**

* Welcome header with overview text
* KPI cards (Inbox, Journey, Data Enrichment)
* Conversation topic analysis (segmented bar)
* Expandable breakdown rows
* Insight callout card
* Top conversion paths card
* Action required panel (alerts & priorities)
* Journey snapshot visualization

**Purpose:**
Quick overview of activity, performance, and next actions.

---

### 2. Inbox / Messages

**Layout:**

* Left: conversation list with search, filters, tabs
* Center: conversation thread
* Right: customer info panel

**Features:**

* Customer header with profile + tags
* AI assistant status indicator
* Takeover Chat / Call buttons
* Mixed timeline (messages + AI + calls)
* Call transcript player with timeline
* Message input with quick replies
* Customer info panel:

  * Profile + Journey tabs
  * Contact details
  * Location map
  * Tags
  * Intent & signals
  * Data enrichment module

**Purpose:**
Central workspace for communication + customer intelligence.

---

### 3. User Journeys

**Sections:**

* Header with subtitle
* KPI cards (visitors, conversion rate, entry page, session duration)
* Tabs (Aggregate vs Individual view)
* Filters (date, sources, devices)
* Story vs Explorer toggle
* Journey visualization (flow diagram)
* Nodes (Homepage, Features, Pricing, Sign Up, Exit)
* Drop-off and intent indicators
* AI insights panel
* Suggested actions (priority-based)
* Top paths summary
* Legend

**Purpose:**
Understand user behavior and optimize conversion paths.

---

### 4. Data Enrichment / Visitor Enrichment

**Sections:**

* Header with summary + timestamp
* Export button
* Credit usage cards
* Fair usage info
* Credit unlock options (email, role, etc.)
* Add contact CTA
* Search + toolbar (filters, export, import)
* Data table:

  * Name
  * Contact
  * Status
  * Last activity
  * Enrichment state
  * Source
  * Actions
* Row states (enriched, partial, missing, locked)
* Source icons + unlock buttons
* Right-side filter drawer:

  * Source
  * Status
  * Date filters
  * Enrichment status
  * Buying intent
  * Chips, checkboxes, collapsible groups

**Purpose:**
Manage and enrich leads efficiently at scale.

---

### 5. Chat Widget Management

#### 5.1 Empty State

**Sections:**

* Page header (Chat Widget Management)
* Empty state illustration/icon
* Message: “No widgets yet”
* Supporting description
* Primary CTA: Create Widget

**Purpose:**
Guide first-time users to create their first widget.

---

#### 5.2 Widget List

**Sections:**

* Search bar
* Status filter dropdown
* Create Widget CTA (top right)
* Widget cards grid:

  * Widget name
  * Created date
  * Status toggle (active/inactive)
  * Settings icon
  * Edit button

**Purpose:**
Overview and quick management of all widgets.

---

#### 5.3 Manage Widget

**Sections:**

* Header with widget name + back navigation
* Plan card (Free / Conversational upgrade)
* Step-based setup:

  1. Connect Website

     * URL input field
     * Scan Website button
     * Progress indicator
  2. Customize Widget (collapsible section)
* Live Preview panel (right side):

  * Browser-style container
  * Chat UI preview
* Upgrade banner (voice / conversational upsell)
* Quick Actions panel:

  * Status toggle
  * Publish button
  * Deactivate button
  * Copy Code
  * Open widget

**Purpose:**
Configure, preview, and deploy chat widgets.

---

#### 5.4 Widget Settings

**Sections:**

* Tabs:

  * Personality & Greetings
  * Voice (locked)
  * Language (locked)
  * Integrations (locked)
  * Analysis (locked)
  * Tools (locked)
  * Advanced (locked)
* Upgrade banner for advanced features
* Form fields:

  * Assistant name
  * Greeting message
  * Personality prompt
* Live preview chat panel (right side)
* Actions:

  * Save & Preview
  * Publish

**Purpose:**
Define AI assistant behavior, tone, and responses.

---

### 6. Train customers.new

**Sections:**

* Header with subtitle
* Auto-scanned website summary (pages count)
* Rescan Website CTA

**Training Methods:**

1. Custom Q&A

   * Add manual Q&A pairs
   * Table of questions and answers
   * Delete/edit actions
   * Add Q&A button

2. Upload Documents

   * Upload files (procedures, policies, etc.)
   * File ingestion for training

3. Talk to customers.new

   * Conversational training interface
   * Start training chat CTA

**Additional Sections:**

* Upgrade banner for advanced training features
* Preview & Test panel (chat simulation)

**Purpose:**
Train and improve the AI assistant using structured and unstructured data.
